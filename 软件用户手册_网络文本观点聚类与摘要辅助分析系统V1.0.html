网络文本观点聚类与摘要辅助分析系统 V1.0 说明文档

一、软件概述

1. 开发目的

本系统用于对网络文本进行集中管理、任务化观点聚类分析与摘要结果管理，形成从“文本录入”到“聚类处理”再到“摘要输出”的闭环处理流程。系统定位为业务处理与信息整理工具，不包含预测、预警、决策推理或知识图谱构建能力。

2. 面向对象

系统面向需要处理多条文本并输出聚类结果与摘要结果的业务人员、分析人员与维护人员。系统通过统一的 Web 前后端架构提供页面操作与接口访问。

3. 主要功能

本系统围绕“文本数据处理→聚类任务执行→摘要结果管理”提供完整业务功能。首先，在认证环节提供注册、登录、登出，并通过 Bearer Token 维护登录态与接口访问权限。其次，在文本管理环节支持文本新增、分页查询、按关键词检索、按分类筛选、单条编辑、单条删除、批量删除，页面端以列表+弹窗方式完成录入与维护，服务端按当前用户归属校验数据可见性与可操作性。再次，在聚类分析环节支持任务创建（name、description、textIds、clusterCount、algorithm）、任务列表查询、任务详情查看、任务执行与任务删除；任务执行后会写入 cluster_results，并将状态从 pending/running 更新到 completed，形成可追踪的任务处理记录。在摘要环节，系统支持按来源生成摘要（sourceType=text/cluster，sourceId），并提供摘要分页查询、单条详情、删除等管理能力；生成后的标题、内容、字数等信息可在摘要管理页面继续维护。系统还提供仪表盘与个人中心功能：仪表盘展示文本总数、聚类任务数、摘要数量、任务状态分布、文本分类分布、近7天趋势与最近操作；个人中心支持资料更新、密码修改、使用统计查看；操作历史模块支持按 type、keyword 检索与分页查询。上述功能均由已实现前端页面、后端接口与数据库字段直接支撑。

4. 技术特点

系统采用 Vue3+Vite 与 Flask 前后端分离架构，统一使用 Bearer Token 鉴权与 {code,message,data} 响应格式；数据默认持久化到 SQLite，并通过服务层完成文本、聚类、摘要与历史记录的业务解耦。

三、系统总体说明

1. 系统结构

系统由前端展示层、后端接口层、业务服务层、数据访问层与数据存储层构成：

  • 展示层：Vue 页面与通用组件，负责数据录入、列表展示、图表展示、弹窗交互。
  • 接口层：Flask Blueprint，负责参数接收、权限验证、调用服务层并返回统一响应。
  • 服务层：处理文本、聚类、摘要、用户、历史等业务规则。
  • 数据层：database.py 提供连接、执行与查询封装。
  • 存储层：SQLite 表（users、auth_tokens、texts、cluster_tasks、cluster_results、summaries、operation_history）。

2. 模块组成（对应真实路由）

3. 数据流转

您登录成功后，系统会在当前浏览器会话中记住登录状态，之后打开各功能页时一般无需反复登录；业务数据始终按当前账号隔离。

文本管理模块负责录入与维护原文；聚类分析模块在选定文本后执行任务并产出簇结果。摘要生成模块仅在「摘要生成」页面完成一次生成动作；生成结果进入库后，由摘要管理模块在「摘要管理」页面独立承担检索、查看与删除。操作历史模块单独记录各类关键动作，供「操作历史」页面查询；个人中心模块的统计与图表仅展示在个人中心页，与历史列表互不混写。

四、核心功能模块说明

1. 认证模块（登录、注册、登出）

模块作用：完成用户身份建立与会话有效性维护。

页面组成：登录页（用户名、密码）、注册页（用户名、密码、昵称、邮箱）。

主要操作流程：在注册页依次填入用户名、密码、昵称、邮箱等信息后，点击注册按钮；校验通过即完成账号创建，随后可返回登录页。在登录页填入用户名与密码，点击登录按钮；验证成功后系统会记住登录状态并自动进入系统首页（仪表盘）。已登录时点击登出，登录状态会被清除，页面回到登录入口。

系统反馈：参数缺失或校验失败返回 400，认证失败返回 401，成功返回 200。

相关数据项：user.id、username、nickname、email、role、createdAt、token。

2. 文本管理模块

模块作用：维护文本基础数据，为聚类与摘要提供源数据。

页面组成：筛选区（keyword、category）、数据表、分页、新建/编辑弹窗、删除确认框、批量删除确认框。

主要操作流程：进入文本管理页后，列表会自动加载当前账号下的文本。需要筛选时，在关键词或分类条件中填入或选择内容，再点击查询，即可得到匹配的列表与分页结果。要新增文本，点击新建按钮，在弹窗中填写标题、正文、分类等必填项后保存，成功后列表会出现新记录。修改某条时点击编辑，在弹窗中改好再保存即可更新。删除单条时点删除并在确认框中确认；需要批量清理时勾选多条后点批量删除，确认后即一并移除。

系统反馈：创建成功返回新增记录；查询返回 list/total/page/pageSize；无权限或不存在返回 404。

相关数据项：id、title、content、word_count、category、status、created_at、updated_at（前端映射为驼峰字段）。

3. 聚类分析模块

模块作用：管理聚类任务并生成任务结果。

页面组成：任务筛选区（keyword、status）、任务卡片列表、新建任务弹窗、删除确认框、结果页图表区。

主要操作流程：在聚类分析页可先按关键词或任务状态筛选，列表以任务卡片形式展示。点击新建任务，在弹窗中填写任务名称、说明，勾选要参与聚类的文本、设定簇数量与算法等，保存后即生成一条待执行任务。对该任务点击执行，系统会依次进入“排队/运行/完成”等状态；完成后可进入该任务的详情或结果页，查看各簇划分与图表展示。若不再需要某任务，点删除并在确认后移除即可。

系统反馈：创建后初始状态为 pending；执行时更新为 running 后转为 completed 并写入 cluster_results；错误时返回明确 message。

相关数据项：name、description、text_ids、cluster_count、algorithm、status、progress、completed_at、results。

4. 摘要生成模块

模块作用：在独立页面中，按用户选定的来源（单条文本或某聚类任务的某一簇）触发一次摘要生成，并将结果写入系统。

页面组成：摘要标题输入、来源类型与具体来源选择（文本列表或任务与簇）、生成按钮、生成完成后的正文预览区。

主要操作流程：打开「摘要生成」页面后，先为本次摘要填写标题；再选择来源——若选单条文本，则在列表中点选一条已保存的原文；若选聚类结果，则先选定聚类任务，再选定要参与摘要的具体簇。必填项齐全后点击生成按钮，等待处理结束；成功后当前页会展示本次生成的摘要正文预览。本页职责到「生成并预览」为止，不在此页维护历史列表。

系统反馈：生成成功会返回一条摘要记录（含标题、正文、字数等）；若来源未选全、来源无效或无权访问，页面会提示相应错误信息。

相关数据项：生成入参侧为 sourceType、sourceId、title；落库侧含 source_type、source_id、title、content、word_count、created_at 等。

5. 摘要管理模块

模块作用：在独立页面中，对已入库的摘要做检索、查看与删除，不负责再次调用生成算法。

页面组成：搜索框、摘要卡片列表、卡片内或展开区的详情展示、单条删除及确认交互。

主要操作流程：打开「摘要管理」页面后，列表会加载当前账号下的历史摘要。需要定位某条时，在搜索框输入关键字并触发查询。点击某张卡片或展开操作，可阅读该条摘要的完整信息与来源说明。若某条不再需要，点击删除并在确认框中确认，确认后该条从列表移除。需要新生成摘要时，应前往摘要生成模块对应页面，不在本页发起生成。

系统反馈：列表与分页查询成功返回多条摘要的列表数据；详情成功返回单条完整字段；删除成功返回成功提示，记录不存在或无权限时返回错误说明。

相关数据项：列表与详情共用 id、source_type、source_id、title、content、word_count、created_at 等；列表接口另含分页字段 list/total/page/pageSize。

6. 个人中心模块

模块作用：维护当前登录用户的昵称、邮箱等资料，修改登录密码，并在本页展示与个人相关的用量统计与图表。

页面组成：个人资料展示与编辑区、密码修改表单、统计数字卡片、雷达图或活跃度等图表区域。

主要操作流程：进入「个人中心」页面后，资料区展示当前昵称、邮箱等；修改后点击保存，校验通过即更新资料。修改密码时，在新密码与确认密码两处填写一致内容后提交，通过后即完成密码更换。页面中的统计卡片与图表在打开或刷新本页时加载，用于查看文本量、任务量、摘要量及活跃趋势等。本页不提供按条操作流水检索，该项由操作历史模块承担。

系统反馈：资料读取与更新、统计类接口成功时返回结构化数据；旧密码或新密码校验失败时返回明确错误消息。

相关数据项：nickname、email 等资料字段；textCount、clusterCount、summaryCount、usageDays、activityData、recentActivity 等统计字段。

7. 操作历史模块

模块作用：按条件分页查询系统自动记录的操作流水，便于审计与自查；不在此页修改密码或编辑个人资料。

页面组成：操作类型筛选、关键字筛选、时间序列表格或列表、分页控件。

主要操作流程：打开「操作历史」页面后，可按操作类型、关键字设置筛选条件，再点击查询或翻页，浏览符合条件的记录。本页仅查看与检索，不进行文本录入、聚类执行或摘要生成。若需看汇总图表，应前往个人中心或仪表盘对应页面。

系统反馈：查询成功返回分页列表；筛选条件不合法或无数据时，列表为空或提示无匹配记录。

相关数据项：history.list（单条含类型、描述、时间等）、分页参数及筛选条件字段。

五、关键处理流程说明

1. 认证与访问控制流程

您完成登录后，再访问系统内各菜单时，系统会自动带上身份校验；若凭证失效或未登录，页面会提示并跳回登录页，避免误操作他人数据。

2. 文本到聚类结果流程

先在文本管理中把待分析内容准备好并保存成功，再到聚类分析里新建任务、勾选这些文本并填好簇数等参数。点击执行任务后，请稍候直至任务状态变为完成；此时打开该任务的结果页，即可看到系统为各条文本分配的簇与可视化结果。

3. 摘要生成流程（摘要生成模块）

在摘要生成页面完成：先填摘要标题，再选择来源——选“单条文本”时指定一条已保存原文，选“聚类任务”时指定任务与具体簇；最后点击生成并等待结束。成功后当前页展示本次生成结果预览，记录已写入系统。

4. 摘要维护流程（摘要管理模块）

在摘要管理页面完成：进入后浏览或按关键字搜索已生成的摘要；点开某条查看详情；若需清理，则对该条执行删除并确认。本流程不触发新的摘要生成，新生成须回到摘要生成页面操作。

5. 仪表盘统计流程

打开首页（仪表盘）时，页面会自动拉取与您账号相关的汇总数据：文本总量、聚类任务数量与状态分布、文本分类分布、近 7 天活跃趋势以及最近操作摘要等，并以数字卡片和图表形式呈现，便于一眼了解当前使用情况。

6. 操作历史记录流程

当您在系统中完成文本新增或编辑、创建/执行聚类任务、生成摘要等操作时，这些动作会被自动记一笔流水。之后进入操作历史页，通过类型或关键字筛选并翻页，即可按时间顺序回顾自己都做过哪些事。

六、软件特点说明

  • 实现范围明确：功能集中在文本管理、聚类任务处理、摘要生成与用户统计，模块边界清晰。
  • 前后端字段映射一致：后端下划线字段通过接口层统一转换为前端使用字段，降低页面处理复杂度。
  • 接口规范统一：统一响应结构、统一鉴权方式、统一分页参数，便于扩展与维护。
  • 业务可追踪：关键操作写入历史表，可用于页面展示与近 7 天活跃数据统计。
  • 部署要求低：默认依赖 Python 标准库数据库组件与 Flask 核心库，可快速部署与本地运行。

文档依据当前项目源码（前端路由、页面组件、后端接口与服务实现）编写，内容范围与已实现能力一致。
